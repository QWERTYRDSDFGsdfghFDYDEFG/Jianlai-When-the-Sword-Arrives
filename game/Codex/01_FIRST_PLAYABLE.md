# 01_FIRST_PLAYABLE - 第一版可玩原型

## Codex 执行前置要求

你正在开发 Jianlai-When-the-Sword-Arrives v1.0 项目《剑来—书简湖》。请先阅读：

1. `01_FIRST_PLAYABLE.md`
2. `GDD.md`

你的任务不是自由发挥做一个新游戏，而是在现有设计约束内，把项目逐步实现成可运行原型。

强制要求：

- 先保证可运行，再逐步增强表现。
- 不要一次性重写全部目录。
- 每次提交都要说明改了哪些文件、实现了什么、如何验证。
- 发现设计不清楚时，先在 `Docs/OPEN_QUESTIONS.md` 记录，不要擅自改核心方向。

## 项目定位

本项目面向 Steam PC 上架，商业模式为买断制，表现形式为 2D 叙事游戏 / 视觉小说。第一版可玩原型的目标不是完成全部内容，而是验证玩家从启动游戏到完成一段完整体验的基本流程是否成立。

## 阶段目标

做出一个可以交给外部测试者游玩的最小版本。玩家应当能够正常启动游戏、进入主线、阅读剧情、看到关键画面和角色、听到基础音频、进行必要选择、保存/读取，并到达一个明确的阶段结束点。

第一版可玩应优先证明：

- 游戏可以稳定启动和退出。
- 主菜单、设置、存档、读档等基础流程可用。
- 第一段主线内容可以从头到尾无报错游玩。
- 文字、角色、背景、音乐、音效的呈现方式达到商业项目的基础观感。
- 玩家能够理解当前故事目标、角色关系和下一步期待。

## 核心可玩范围

### 1. 启动与主菜单

任务：

- 确认游戏可从 Ren'Py 启动器正常运行。
- 主菜单包含开始游戏、读取游戏、设置、关于/制作名单、退出。
- 新游戏进入 `label start` 后没有脚本错误。
- 退出流程在窗口模式和全屏模式下都正常。

完成标准：

- 连续启动 3 次无崩溃。
- 主菜单按钮均可点击。
- 设置、读取、退出不会进入死循环或空白界面。
- 没有明显缺图、缺字、缺音频报错。

### 2. 第一段主线体验

任务：

- 确认第一章或第一段核心剧情可以完整游玩。
- 明确本阶段的结束点，例如进入第二章、出现章节结束提示，或回到主菜单。
- 检查所有角色名、旁白、对白格式一致。
- 检查文本没有明显占位符、乱码、重复行、断句异常。
- 检查关键剧情节点有合适的背景或角色立绘支撑。

完成标准：

- 从新游戏开始到阶段结束点，全程不需要开发者干预。
- 玩家能理解主角当前处境、主要冲突和继续游玩的动机。
- 没有 `TODO`、临时调试文本、未替换素材名暴露给玩家。
- 阶段结束后玩家知道自己已经完成了当前测试内容。

### 3. 选择与分支

任务：

- 如果当前版本包含选项，确认每个选项都能进入有效结果。
- 选项文案应表达清楚，不使用开发者占位描述。
- 关键选择需要有可感知反馈，例如对白变化、变量变化、角色态度变化或后续文本变化。
- 如果暂时不做真正分支，至少保证选项不会破坏主线流程。

完成标准：

- 每个选项至少测试 1 次。
- 不存在选择后黑屏、跳错章节、提前 `return`、无法继续的情况。
- 玩家不会误以为某个未完成分支是正式内容。

### 4. 视觉呈现

任务：

- 检查第一版需要用到的背景图是否齐全。
- 检查角色立绘、表情、位置和切换是否自然。
- 检查 UI 字体、字号、文本框、姓名框在 1080p 下清晰可读。
- 检查窗口模式和全屏模式下画面没有严重拉伸、裁切或错位。
- 对明显临时素材进行标记，列入后续美术资产清单。

完成标准：

- 主要剧情场景不会长时间停留在空背景或默认占位图。
- 角色名、对白、菜单文字没有被遮挡。
- Steam 截图级别的关键画面至少有 3 个可用候选。
- 临时素材不会被误认为最终质量。

### 5. 音频体验

任务：

- 第一段主线至少包含基础背景音乐。
- 关键场景可加入必要音效，例如门、脚步、剑声、环境声、确认按钮声。
- 设置菜单中的音乐、音效音量可调整。
- 检查音频循环、停止、切换是否突兀。

完成标准：

- 音乐不会在章节切换后异常叠加。
- 音量设置生效，且保存后重新进入仍可用。
- 没有刺耳爆音、过大音量、明显不匹配的音效。

### 6. 存档与读档

任务：

- 测试自动存档、手动存档、读档、快速存档、快速读档。
- 确认关键剧情节点读档后变量、背景、角色状态正确。
- 检查旧存档是否会导致当前版本报错，如会报错，需要记录。

完成标准：

- 第一段主线中至少 3 个不同位置可以正常存档和读档。
- 读档后不会丢失角色立绘、背景、音乐或剧情变量。
- 存档缩略图、时间、页码显示正常。

### 7. PC 操作与设置

任务：

- 确认鼠标、键盘 Enter、Space、Esc、Ctrl 快进等 Ren'Py 默认操作可用。
- 确认窗口/全屏切换可用。
- 确认文字速度、自动前进、跳过设置可用。
- 确认中文显示稳定，字体不会缺字。

完成标准：

- 玩家不用说明书也能完成第一段体验。
- Esc 或右键可稳定打开游戏菜单。
- 快进不会跳过到异常位置或破坏演出。
- 1080p 显示下主要 UI 可读，窗口缩放后无严重错位。

## 商业发布视角的最低验收

第一版可玩完成时，应满足以下条件：

- 无启动崩溃。
- 无阻断主线的脚本错误。
- 无明显缺失的核心素材。
- 无无法退出、无法读档、无法继续的严重流程问题。
- 第一段体验有明确开头、推进和结束点。
- 玩家能在 10 到 30 分钟内完成一次测试流程。
- 测试者可以用一句话说清楚游戏是什么、主角遇到了什么、为什么要继续玩。

## 测试流程

每次提交第一版可玩前，按以下顺序测试：

1. 删除或移走临时测试存档后启动游戏。
2. 从主菜单开始新游戏。
3. 正常速度完整阅读第一段主线。
4. 中途手动存档一次并立即读档。
5. 中途退出到主菜单，再读取存档继续。
6. 测试一个选项或关键互动点。
7. 快进一小段已读文本，确认不会卡死。
8. 到达阶段结束点。
9. 返回主菜单并正常退出游戏。

## 不纳入本阶段的内容

以下内容重要，但不作为第一版可玩的必要完成项：

- 全章节完整文本。
- 完整 Steam 成就系统。
- Steam 云存档。
- 商店页全部美术素材。
- 完整配音。
- 全量 CG 和最终立绘。
- 多语言本地化。
- 完整发布包和安装器。

这些内容应在后续任务文档中拆分，例如 UI/设置/Steam 准备、美术资产、发布验收清单等。
