# 角色出图工作流

用途：解决三个高频问题：

- 换聊天后，同一角色身份漂移
- 换表情但看起来几乎没区别
- 动作僵硬，像站桩或摆拍

配套规则：

- `Codex/character_asset_update_rules.md`
- `Codex/naming_guidelines.md`

---

## 1. 为什么换聊天后会不像同一个人

核心原因通常不是“母版没用”，而是下面三件事同时发生：

- 新聊天不会继承上一轮对角色的隐性记忆，只会遵守当前轮明确提供的图和文字约束。
- 如果母版主要是全身图，脸部在画面里占比太小，模型更容易抓住“服装和大轮廓”，抓不稳脸。
- 如果提示词里只写“同一个人、锁脸、基于母版”，但没有把五官比例、年龄感、气质底色、服装结构重新明确写一遍，模型会自动补成“像这一类人”，而不是“就是这个人”。

结论：每次进入新聊天，都必须把“锁身份包”重新完整喂一遍，不能只说“沿用上一版”。

---

## 2. 新聊天强制启动包

只要是正式角色出图，默认都要同时提供这四类信息：

1. 角色专属标准文档
2. 当前唯一母版图
3. 角度辅助图 / 脸部辅助图
4. 本次只允许变化的变量清单

推荐固定写法：

```text
本次不是重新设计角色，而是基于既有母版做同一人物的稳定变体。
必须锁定：脸型、五官比例、年龄感、发型轮廓、身形体态、气质底色、当前服装结构。
本次只允许变化：[表情] / [动作] / [场景] / [景别] / [轻微光线]。
如果新图更好看但不像母版里的同一个人，直接判定失败。
```

如果角色只有全身母版，没有单独脸部近景母版，必须额外把脸部识别点重新写出来，不能偷懒。

---

## 3. 三阶段生成法

不要一上来就做“新表情 + 新动作 + 新镜头 + 新场景”。正确顺序是：

### 第一阶段：先锁身份

目标：确认“还是同一个人”。

- 优先中近景或半身
- 正常光线
- 中性或轻表情
- 不做大动作

只要这一阶段都不稳，后面所有变体都会继续漂。

### 第二阶段：再做表情差分

目标：在不换人的前提下，让表情真的变。

- 镜头尽量先不变
- 动作尽量先不变
- 先只改脸部和头部小角度
- 一次只做一个情绪方向

### 第三阶段：最后做动作差分

目标：在身份稳定、表情稳定以后，再让身体活起来。

- 先小动作，再中动作
- 先单人站姿变体，再做互动和大幅运动
- 先把重心和线条做清楚，再追求戏剧张力

---

## 4. 表情不要再写成标签

以后不要只写：

- `认真`
- `疑惑`
- `委屈`
- `似笑非笑`

这种写法太抽象，模型很容易全部画成同一张轻微变体脸。

表情必须至少拆成 5 个可见变量：

1. 眉
2. 眼睑 / 眼神
3. 嘴角 / 唇形
4. 面颊 / 下颌张力
5. 头部微角度

通用写法模板：

```text
表情不是泛泛的[情绪名]，而是：
眉：[具体变化]
眼睛：[具体变化]
嘴：[具体变化]
面颊和下颌：[具体变化]
头部：[具体变化]
```

示例：

```text
认真：眉头轻收但不拧紧，上眼睑略压，目光集中，嘴角收平，下颌轻收，头微微前倾。
疑惑：一侧眉略抬，双眼轻微收窄，视线带停顿感，嘴微张但不夸张，头轻偏一侧。
委屈：眉头向中间轻靠但克制，下眼睑略提，嘴角轻向下收，面颊有一点绷住的感觉，头略低。
似笑非笑：眉眼放松但带观察感，眼尾略收，嘴角单侧极轻上提，下颌放松，头微偏。
```

---

## 5. 动作不要再写成名词

以后不要只写：

- `站立`
- `回头`
- `走路`
- `指人说话`

这种写法会直接生成“默认站桩姿势”。

动作必须至少拆成 7 个变量：

1. 重心在哪条腿
2. 另一条腿在做什么
3. 骨盆是否偏转
4. 肩线是否倾斜
5. 左手在做什么
6. 右手在做什么
7. 头朝向哪边、转多少

有余量时，再补两项：

8. 躯干主线条是直、弧还是前倾
9. 衣摆 / 发尾 / 袖口有没有轻微跟随

通用写法模板：

```text
身体姿态：[正身 / 半侧身 / 轻转身]
重心：[左脚 / 右脚 / 平均]
双腿：[支撑腿状态] + [自由腿状态]
肩和腰：[是否有轻微倾斜或转动]
左手：[位置和动作]
右手：[位置和动作]
头部：[朝向和幅度]
衣摆与发丝：[是否有轻微跟随]
```

示例：

```text
半侧身站立，重心落在左脚，右脚向后撤半步；骨盆轻微右转，肩线保持放松；左手自然垂落，右手抬到胸口下方轻指前方；头向右回望约 25 度；衣摆和发尾有轻微跟随，不要僵直站桩。
```

---

## 6. 去僵硬的硬规则

- 一张图里只允许一个主动作，不要让手脚腰头同时抢戏。
- 先做“不完全对称”，再谈“好看”。
- 支撑腿必须稳定，自由腿必须有任务，不能双腿一样直。
- 手势必须有高低差，不能左右手都贴在身体两侧像待机。
- 头转了，发型和视线也必须跟着转；身体转了，肩线和衣摆也要跟。
- 有动作就要有重心；没有重心的动作，基本都会僵。
- 要求“自然”时，不要再同时要求“正面、全身、完全站直、双手都清楚、脸也最标准”，这几个要求叠在一起最容易变假。

---

## 7. 正式生产顺序

推荐每个角色都按这个顺序积素材：

1. 脸部锁定近景母版
2. 通用站姿全身母版
3. 同镜头表情差分
4. 小动作差分
5. 中动作差分
6. 特定场景演出图

如果缺第 1 步，后续换聊天时最容易漂。

---

## 8. 验收时优先看什么

验收顺序固定如下：

1. 第一眼是否还是那个人
2. 脸部比例有没有漂
3. 年龄感有没有漂
4. 气质底色有没有换人
5. 服装结构有没有跑偏
6. 表情是否真的与上一张不同
7. 动作是否有明确重心、方向和跟随关系

只要前 1 到 5 项不稳，就不要因为“这张更好看”而留下。

---

## 8.5 不要每次出图都改文档

出图和改角色标准文档，不是同一个动作。

默认节奏应当是：

1. 先出测试图
2. 再筛候选图
3. 只有正式替换当前母版时，才更新标准文档

也就是说：

- 新图和旧图不一样，不等于要改文档
- 这张图更顺眼，不等于要改文档
- 不同人生成结果略有差异，不等于要改文档

只有当用户明确确认“这张以后作为新的正式母版 / 正式表情页 / 默认参考页”时，才回写文档。

详细规则见：

- `Codex/character_asset_update_rules.md`

## 9. 当前项目的直接结论

- 陈平安更稳，关键原因不是碰巧，而是他已经接近“脸部母版”和“服装母版”分离的工作方式。
- 李宝瓶和崔东山现在虽然也有母版和标准文档，但仍更依赖“全身母版 + 抽象描述”，所以在新聊天里更容易漂。
- 后续如果要继续提高李宝瓶、崔东山的稳定度，优先级最高的补强不是继续堆长提示词，而是补“脸部近景锁定母版”与“固定表情差分页”。
