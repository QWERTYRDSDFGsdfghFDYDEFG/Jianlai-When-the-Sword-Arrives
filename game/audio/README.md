# 音频资源管理约定（建议）

现在 `audio/` 里已经是“章节 1 的配音 + 少量 BGM”，后续章节会指数增长。建议尽早把“文件组织 + 引用方式 + 校验工具”定下来，避免后面改动成本爆炸。

## 1) 目录结构（推荐）

把音频按用途分根目录（比按章节分更稳定）：

```
audio/
  bgm/          # 背景音乐
  amb/          # 环境氛围（风声、人群、雨声…）
  sfx/          # 音效（脚步、拔剑、UI 点击…）
  ui/           # UI 声音（按钮、翻页…）
  voice/        # 配音（随章节增长的主要部分）
```

配音再按“章节/场景/角色”分层（选一种就行，保持一致）：

方案 A（按章节）：`audio/voice/ch01/<角色>/<行号>.ogg`
方案 B（按场景）：`audio/voice/scn_<场景ID>/<角色>/<行号>.ogg`

> 更推荐“按场景”的方案：将来章节拆分、补写、重排剧情时，场景 ID 更稳定。

## 2) 命名规范（建议）

为了跨平台与工具兼容，建议逐步把文件名从中文+空格迁移到 ASCII（保留一份映射表即可）：

- 角色：`cpy`(陈平安) / `lbp`(李宝瓶) / `cds`(崔东山) / `narr`(旁白) …
- 场景：`scn_0001_midnight_courtyard`
- 行号：`0001` 起步，方便插入补录

示例：

```
audio/voice/scn_0001_midnight_courtyard/lbp/0001.ogg
audio/bgm/theme_main.ogg
audio/sfx/sword_draw_01.wav
```

## 3) 引用方式（建议）

短期（现在）：`voice "audio/xxx.mp3"` 没问题，但会越来越难维护（改名/移动必改脚本）。

中期（推荐）：引入“索引表”，脚本里只引用 **ID**，文件移动/格式替换不改剧情脚本：

- 在 `audio/_index.json`（或 `audio/_index.csv`）里维护：`voice.scn_0001.lbp.0001 -> 路径`
- 让工具脚本自动生成/校验这个索引

本仓库已提供一个校验脚本：`tools/audio_audit.ps1`（见下文）。

## 4) 每次新增音频的流程（建议）

1. 把文件放进对应目录（先不强制改名也行，后续再迁移）。
2. 运行 `tools/audio_audit.ps1` 看是否：
   - 有脚本引用了不存在的文件
   - 有音频文件完全没被引用（可能是漏用或废弃）
3. 逐步把“章节 1”的中文文件名迁移到规范目录/命名（建议一次只迁移一个角色或一个场景）。
