# script.rpy 逐场分镜表

> 用途：将 `script.rpy` 的序章内容拆成可制作的 VN 分场演出表。  
> 目标：先把 `script.rpy` 的重做图片需求落到“背景、立绘、表情、站位、景别、演出备注”，再进入正式出图。  
> 口径：优先复用场景基底，控制新图数量，保持《剑来》这章的留白、送别、夜色与人情。

---

## 使用规则

- 全部新图默认按 `1920 x 1080` 横版出图。
- 本表按 `script.rpy` 里每一次 `scene bg` 切换拆场，方便后续一图对一场。
- `背景` 优先按“主图 + 派生图”组织，而不是每场都完全重画。
- `立绘` 先写成建议占位，后续可以按真实素材名替换。
- `表情` 先写大类，不提前细拆到过多差分。
- `景别` 以 `远景 / 中景 / 近景 / 特写` 为主，VN 里主要靠裁切和缩放完成。

---

## 序章图片重做总策略

- 当前 `script.rpy` 共用到 `14` 个背景切点，建议整理为 `9` 张主图 + `5` 张派生图。
- 最值得统一重做的是“书院夜色组”“湖边演出组”“陈平安演剑组三大套”。
- `c1_02_midnight_lake_v1 / c1_03_lh_entry_v1 / c1_06_duet_lake_v1 / c1_07_pq_run_v1 / c1_08_pq_pose_v1 / c1_09_platform_duet_v1` 建议共用同一湖区夜景世界观，只在构图、人物和金色法术特效上区分。
- `c1_04_stage_far_v1 / c1_05_stage_near_v1` 建议视为同一套戏台演出图的远近景版本。
- `c1_10_cpa_arrival_v1 / c1_11_cpa_sword_01_v1 / c1_12_cpa_sword_02_v1` 建议视为同一场“白衣剑仙登场”的三段式主视觉，而不是三张互不相关的图。

## 生成图片前的准备工作

- 先把每个场次的 `统一主名` 与当前 `script.rpy` 中现用 `scene bg` 做对应，确认是否可以用占位图继续推进脚本。
- 先根据本表判断每张图的性质：主图 / 派生图 / 纯切分图。优先把 `c1_02_midnight_lake_v1`、`c1_04_stage_*`、`c1_10_cpa_*` 这几组母图规划出来。
- 先明确“本场是否需要独立立绘”：若场次可靠整图推进，则暂用整图占位，避免提前拆差分；若需要对话立绘，则先补占位立绘入场代码。
- 先建立“现用图名 / 统一主名 / 复用关系 / 资源状态”清单，减少以后反复重画同一构图的风险。
- 先把本表里的场次拆成“可以写成脚本结构”的样子，再交给图片制作；不要先出图再改场次逻辑。
