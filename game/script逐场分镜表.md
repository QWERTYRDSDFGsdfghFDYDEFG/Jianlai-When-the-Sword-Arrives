# script.rpy 逐场分镜表

> 用途：将 `script.rpy` 的序章内容拆成可制作的 VN 分场演出表。  
> 目标：先把 `script.rpy` 的重做图片需求落到“背景、立绘、表情、站位、景别、演出备注”，再进入正式出图。  
> 口径：优先复用场景基底，控制新图数量，保持《剑来》这章的留白、送别、夜色与人情。

---

## 使用规则

- 全部新图默认按 `1920 x 1080` 横版出图。
- 本表按 `script.rpy` 里每一次 `scene bg` 切换拆场，方便后续一图对一场。
- `背景` 优先按“主图 + 派生图”组织，而不是每场都完全重画。
- `立绘` 先写成建议占位，后续可以按真实素材名替换，但不能再把所有场次都理解成“普通站姿立绘”。
- `表情` 先写大类，不提前细拆到过多差分。
- `景别` 以 `远景 / 中景 / 近景 / 特写` 为主，VN 里主要靠裁切和缩放完成。
- 逐场出图时，统一先按 [《剑来》VN演出与构图总手册.md](/E:/新建文件夹/When%20the%20Sword%20Arrives/game/《剑来》VN演出与构图总手册.md:1) 判断是 `基础立绘`、`动作立绘` 还是 `事件构图图`。
- 逐场动作和摆位统一参考 [《剑来》VN演出与构图总手册.md](/E:/新建文件夹/When%20the%20Sword%20Arrives/game/《剑来》VN演出与构图总手册.md:1)，不再在每一场里重复整套风格词。

### 本表新增字段说明

- `图像类型`：`基础立绘场 / 动作立绘场 / 事件构图图`
- `动作级别`：`A0 标准静立 / A1 轻动作对白 / A2 关系动作 / A3 事件动作`
- `动作设计`：写清身体重心、手部、脚步、视线
- `禁区`：明确这张图最容易被错误画成什么

### 序章统一风格与禁区

- 风格统一：`东方幻想`、`宋代审美`、`半写实`、`山水志怪`、`人间烟火`、`沉静克制`、`自然情绪光影`、`低饱和高级灰`、`真实材质结构`
- 序章统一禁区：`古偶剧照感`、`仙侠网游风`、`二次元萌系`、`情侣海报感`、`过度唯美滤镜`、`页游特效感`

## 出图前的隐性控制口径

这部分不是替代分镜表，而是给整份表补一层更上游的判断标准。后续每一场出图，都默认先过这三道检查。

### 1. 对 `[物体]` 刻画来说，“透视”的精细度有优先级

- 先画准 `空间骨架`：地面、高台、栏杆、门框、屋檐、桥、湖岸线这类决定空间成立的硬结构，透视必须最稳。
- 再画准 `叙事焦点物`：人物手边、脚下、身上正在使用或被强调的物件，例如竹刀竹剑、葫芦、琴笛、灯笼、法袍下摆、长剑、书箱。
- 最后处理 `远景陪衬物`：远山、树丛、水纹、远处屋脊、散落小物可以概括，不需要和前景关键物用同一级精描。
- 默认优先级：`人物接触物 > 人物附近关键物 > 空间骨架 > 远景装饰物`。
- 不要把所有东西画得一样认真。对 VN 背景来说，真正有用的不是“全都细”，而是“该准的准，该收的收”。
- 只要某个物体承担 `受力`、`方向`、`站位`、`动作起点` 的功能，它的透视优先级就自动上升。

### 2. 对 `[画面]` 构成来说，“块面切分”的安排先于小细节

- 每张图先切 `大块面`，再补 `中块面`，最后才进小装饰；不要一开始就堆碎元素。
- 至少先明确三个层级：`主块面`、`副块面`、`留白块面`。
- `主块面` 负责第一眼读图，比如人物群、高台、湖面主斜线、书院门口主建筑。
- `副块面` 负责托住关系和节奏，比如道路转折、栏杆、树影、岸线、远山。
- `留白块面` 负责呼吸、对白承接、情绪停顿，不能被“怕空”心理填满。
- 前景、中景、后景尽量靠 `明暗`、`虚实`、`冷暖`、`大小` 拉开，而不是只靠堆物件来制造层次。
- 戏多、人多、动作强的场次，更要先把画面切成清楚的大块，不然最后会只剩“热闹”，没有阅读顺序。
- 凡是要兼容 Ren'Py 立绘叠加和对白框的图，中央和两侧都要预留安全块面，避免人物一进场就把画面挤死。

### 3. 对 `[视觉]` 效果来说，“画风”的潜规则比表面风格词更重要

- 画风首先不是笔触花样，而是 `同一世界的空间逻辑 + 光线逻辑 + 材质逻辑 + 人物逻辑` 是否统一。
- 同一组图必须默认共用一套潜规则：镜头高度接近、空气湿度接近、夜色明度区间接近、材质反应接近。
- 《剑来》这套图的“好看”，不是靠仙气滤镜，而是靠 `半写实`、`宋韵审美`、`克制情绪`、`真实材质`、`低饱和高级灰` 一起成立。
- 宁可 `收一点、灰一点、静一点`，也不要 `甜一点、亮一点、艳一点、仙一点`。
- 人物不能偶像化，场景不能样板化，特效不能页游化；一旦出现这三种倾向，哪怕单张图精致，也算跑偏。
- 所谓“统一画风”，本质上是让观者相信这些图发生在同一个人间，而不是每一场都换一个审美系统。

### 4. 三者冲突时的执行优先级

- 优先保证 `空间可信`，其次保证 `块面清楚`，最后才是 `风格修饰`。
- 如果一张图风格很满，但空间站不住、块面读不清，就不能算合格。

## 序章图片重做总策略

- 当前 `script.rpy` 共用到 `14` 个背景切点，建议整理为 `9` 张主图 + `5` 张派生图。
- 最值得统一重做的是“书院夜色组”“湖边演出组”“陈平安演剑组三大套”。
- `c1_02_midnight_lake_v1 / c1_03_lh_entry_v1 / c1_06_duet_lake_v1 / c1_07_pq_run_v1 / c1_08_pq_pose_v1 / c1_09_platform_duet_v1` 建议共用同一湖区夜景世界观，只在构图、人物和金色法术特效上区分。
- `c1_04_stage_far_v1 / c1_05_stage_near_v1` 建议视为同一套戏台演出图的远近景版本。
- `c1_10_cpa_arrival_v1 / c1_11_cpa_sword_01_v1 / c1_12_cpa_sword_02_v1` 建议视为同一场“白衣剑仙登场”的三段式主视觉，而不是三张互不相关的图。

## 生成图片前的准备工作

- 先把每个场次的 `统一主名` 与当前 `script.rpy` 中现用 `scene bg` 做对应，确认是否可以用占位图继续推进脚本。
- 先根据本表判断每张图的性质：主图 / 派生图 / 纯切分图。优先把 `c1_02_midnight_lake_v1`、`c1_04_stage_*`、`c1_10_cpa_*` 这几组母图规划出来。
- 先给每张图补一份简短的 `透视主次 / 块面草图 / 风格禁区` 备注，哪怕只写三行，也比直接进细节可靠。
- 先明确“本场是否需要独立立绘”：若场次可靠整图推进，则暂用整图占位，避免提前拆差分；若需要对话立绘，则先补占位立绘入场代码。
- 先建立“现用图名 / 统一主名 / 复用关系 / 资源状态”清单，减少以后反复重画同一构图的风险。
- 先把本表里的场次拆成“可以写成脚本结构”的样子，再交给图片制作；不要先出图再改场次逻辑。
